# 模块一 用户洞察与问题定义

## 1.1 目标用户

腾讯视频的核心用户群体，尤其是以下特征交叉的人群：

| 特征 | 描述 |
|------|------|
| 年龄 | 18-35岁，覆盖大学生至职场中坚 |
| 行为 | 视频平台重度/中度使用者，追剧、追综是日常娱乐刚需 |
| 心理 | 对广告有本能排斥，但不是拒绝所有商业信息——只是拒绝"和我无关的噪音" |
| 消费 | 有购买力但决策谨慎，需要"被说服"而非"被告知" |

典型用户画像：
- **小陈，24岁**：正在追一部悬疑剧，连续追更型。剧情正到关键时刻，突然一段30秒洗衣液广告弹出——和悬疑氛围完全断裂，只想跳过。
- **李姐，35岁**：带娃间隙看综艺，囤剧一口气型。广告出现时本能拿起手机刷别的，根本没在看。
- **阿杰，28岁**：科技发烧友，看纪录片时遇到美妆广告，毫无兴趣直接闭眼等结束。

## 1.2 用户痛点

我们通过分析视频广告场景，识别出四大核心痛点：

### 痛点一：时长过长，超出耐心阈值
- **数据**：平均前贴片广告时长60秒，而用户对广告的耐心阈值仅15-20秒
- **表现**：用户被迫观看完整时长，无法跳过（非VIP）或需额外付费跳过
- **本质**：时间成本与信息价值严重不匹配——60秒广告可能只有3秒信息与用户相关

### 痛点二：内容无关，信息断裂感强
- **数据**：超过60%的用户选择主动跳过广告
- **表现**：正在看悬疑剧，却弹出零食广告；正在看亲子综艺，却看到汽车广告
- **本质**：广告与用户当前观看内容在情感、主题上完全断裂，造成强烈干扰感

### 痛点三：信息稀释，核心卖点被淹没
- **数据**：前15秒注意力流失率高达45%
- **表现**：广告前半段铺垫冗长，核心卖点直到结尾才出现，但用户早已走神
- **本质**：广告为"所有人"设计，没有重点突出与特定用户最相关的信息

### 痛点四：形式单一，被动接收无掌控感
- **数据**：3倍信息冗余——同样的核心信息用不同话术反复说，反而削弱说服力
- **表现**：广告就是"硬塞"，用户没有任何选择权或参与感
- **本质**：缺少"这个广告为什么和我有关"的第一秒回答，用户处于被动忍受状态

## 1.3 使用场景

### 场景一：追剧间隙的"精粹速览"
小陈正在追一部科幻悬疑剧，剧情紧张时刻进入广告段。传统模式下，他需要忍受一段与悬疑氛围完全断裂的60秒洗护广告。而在**精粹模式**下：
- 系统识别他正在看悬疑内容，心理处于"烧脑紧张"状态
- 洗护广告被AI压缩至30秒精华，2倍速播放
- 同时右侧出现3张个性化便利贴，每张都用他能接受的语言风格（数据驱动、逻辑清晰）呈现产品亮点
- 文案如"**72h净屑** 实验室数据说话，不是空口承诺"——直接回应他对"确定性"的心理需求

### 场景二：综艺间隙的"轻松种草"
李姐在看轻松综艺时进入广告段。精粹模式下：
- 系统识别她处于"轻松欢乐"的情绪，偏好"家庭亲子"类内容
- 同一个产品，AI生成的便利贴文案完全不同："**全家适用** 孩子也能用，放心不下那点事"
- 唤起的是"关爱安全感"心理维度，而非前例的"掌控感与确定性"

**同一个产品，不同的用户，看到完全不同的广告**——这就是AI个性化广告的价值。

# 模块二 产品方案设计

## 2.1 产品概述

**广告精粹**是一个AI驱动的视频广告个性化体验引擎，通过7维心理分析建模用户，实时计算"用户心理 × 产品特性"的共鸣系数，再由大模型生成个性化文案，将传统60秒广告压缩为30秒精粹模式——用户看到的是和自己心理需求最匹配的3个亮点，用他们最听得进去的语言说出来。

## 2.2 核心功能

### 功能一：7维心理画像
在用户观看视频前，系统基于5大维度（账号基础信息、内容偏好图谱、观看行为特征、互动与社交行为、当前情境信号）共13个标签，构建用户的7维心理权重向量：

| 维度 | 代号 | 含义 | 典型触发场景 |
|------|------|------|------------|
| 掌控感与确定性 | P1 | 需要事实、数据、机制来建立信任 | 看纪录片/烧脑内容时 |
| 效率与省力 | P2 | 在乎省时间省麻烦 | 碎片化观看时 |
| 归属感与认同 | P3 | 在意群体认同/圈层归属 | 高互动用户/社交型 |
| 感官愉悦 | P4 | 被感官画面打动 | 轻松解压/猎奇时 |
| 关爱安全感 | P5 | 唤起对家人的情感 | 温馨治愈/家庭亲子 |
| 经济理性 | P6 | 这钱花得值吗 | 非会员/价格敏感型 |
| 自我叙事与成长 | P7 | 符合想象中的自己 | SVIP/追求品质感 |

### 功能二：共鸣系数匹配
为每个产品的每个特性在7个心理维度上打分（人工标注+校准），然后与用户的7维权重向量做加权求和，计算"共鸣系数 R"：

```
R = Σ(W[dim] × Feature.scores[dim])  (dim = P1~P7)
```

选出共鸣系数最高的Top3特性，作为"最可能打动这个用户"的亮点。

### 功能三：LLM实时文案生成
将用户画像、权重分析、Top3亮点送入DeepSeek大模型，生成：
- **个性化特征名**：6字以内，用户友好的功能标签
- **12-18字个性化文案**：精准对应用户的心理触发点
- **心理拆解**：标注每条文案打的是哪个心理维度

### 功能四：精粹模式播放
- 广告视频2倍速播放，时长压缩50%
- 右侧手账风格拼贴板实时展示AI生成的3张便利贴亮点
- 每张便利贴弹入动画，逐个呈现，制造"发现感"

# 模块三 AI原生能力说明

## 3.1 AI核心能力

本产品使用了以下AI能力：

| AI能力 | 应用场景 | 具体实现 |
|--------|---------|---------|
| **消费心理建模** | 用户画像 → 心理权重计算 | 自研7维心理模型，将13标签映射为7维权重向量 |
| **共鸣系数算法** | 产品特性 → 亮点排序 | 加权求和R = Σ(W×S)，数学化匹配用户心理与产品卖点 |
| **大模型文本生成** | 亮点 → 个性化文案 | DeepSeek V4 Flash，结构化Prompt + JSON输出 |
| **情境感知** | 实时调整广告策略 | 基于当前内容情绪、时段、观看深度动态调整权重 |

## 3.2 AI如何解决痛点

| 痛点 | AI解决方案 | 去掉AI还能成立吗？ |
|------|-----------|-----------------|
| 时长过长 | AI压缩时长（2x速）+ 只展示最相关的3个亮点 | ⚠️ 部分成立：2x速不需要AI，但"选哪3个亮点"必须依赖AI |
| 内容无关 | 共鸣系数匹配，选出与用户心理最共振的特性 | ❌ 不成立：没有7维权重+共鸣计算，无法知道"哪些信息和用户相关" |
| 信息稀释 | Top3排序+LLM浓缩为12-18字文案 | ❌ 不成立：没有LLM，无法将产品事实转化为用户能共鸣的精炼文案 |
| 形式单一 | LLM实时生成，每个用户看到不同文案 | ❌ 不成立：模板替换做不出12-18字精准文案，且无法匹配心理触发点 |

**结论**：去掉AI，产品退化为"2倍速播放广告"——压缩了时间但没有解决核心问题（"广告和我有什么关系"）。AI是产品成立的必要条件，不是锦上添花。

## 3.3 AI技术方案

### 模型/API选型

| 组件 | 技术方案 | 选型理由 |
|------|---------|---------|
| 用户画像生成 | 本地随机算法（基于规则） | 13维度标签的组合空间巨大（>10万种），实际生产中对接用户行为数据API |
| 心理权重计算 | 自研算法（基础权重+调整因子+归一化） | 消费心理学理论+经验参数，可解释性强 |
| 共鸣系数计算 | 加权求和排序 | 数学模型，可量化、可调优 |
| 文案生成 | DeepSeek V4 Flash | 性价比高、中文能力强、响应速度快（<2s）、JSON输出稳定 |

### AI工作流程

```
1. 输入阶段
   用户画像(13标签) + 产品数据(特性×7维得分)

2. 计算阶段（本地，<10ms）
   computeUserWeights(画像) → 7维权重 [P1...P7]
   rankFeatures(权重, 产品特性) → 按共鸣系数R降序排列
   从Top5中随机选3个（增加多样性）

3. 生成阶段（DeepSeek API，~1.5s）
   buildCopyPrompt(画像, 权重, Top3亮点, 产品) → 结构化Prompt
   POST /chat/completions → JSON响应

4. 渲染阶段（本地，<50ms）
   解析JSON → 提取featureLabel + copy
   渲染3张便利贴卡片 + 弹入动画
```

### Prompt设计要点

Prompt由4部分组成：
1. **用户画像**：13个维度值 + 对广告策略的指导意义
2. **心理权重分析**：前3位主导维度及百分比
3. **Top3亮点**：产品事实、共鸣系数R、主导心理维度
4. **任务指令**：4步走（深度理解→生成文案→友好特征名→心理拆解）

**关键约束**：
- 文案12-18字，禁止极致/完美/颠覆等词
- 必须有具体场景或可感知细节
- 文案让用户感受"产品整体适合我"，不直接提及画像标签
- 严格JSON输出格式

# 附：方案最大风险与挑战

## 风险一：画像数据获取与隐私
- **问题**：13维度标签需要用户行为数据支撑，涉及隐私合规
- **应对**：画像基于聚合行为而非原始数据，不收集敏感个人信息；画像仅在服务端计算，不暴露给客户端

## 风险二：LLM文案质量控制
- **问题**：大模型偶尔产生幻觉或不合规文案
- **应对**：结构化Prompt约束+JSON格式校验+关键词过滤+降级到预设文案

## 风险三：广告主接受度
- **问题**：广告主可能担心AI改写文案偏离品牌调性
- **应对**：文案生成在品牌框架内（品牌方可设定风格红线），且展示的产品事实经过品牌确认，仅表述方式个性化

## 风险四：三方利益的潜在冲突
- **问题**：用户想要更短/更少广告，平台需要广告收入，广告主需要曝光时长
- **取舍原则**：精粹模式不是"缩短广告"，而是"让广告的每一秒都有效"。时长压缩50%，但信息传达效率提升3倍（3个精准亮点 vs 全量冗余信息）。对广告主而言，用户真正看进去的3秒，胜过走神的60秒。对平台而言，用户体验改善→留存提升→长期广告价值更大。
