## 模块一：用户洞察与问题定义

### 1.1 目标用户

**核心用户画像：**
- 年龄：18-35岁
- 身份：视频平台活跃用户，有一定消费能力
- 行为特征：
  - 习惯性跳过广告（超过60%用户选择跳过）
  - 对无关广告容忍度极低
  - 愿意为自己感兴趣的内容付出注意力

**用户分层：**

| 用户类型 | 特征 | 占比 |
|---------|------|------|
| 效率型 | 追求快速获取信息，厌恶冗余 | 35% |
| 品质型 | 注重体验，愿意为好内容付费 | 25% |
| 社交型 | 喜欢分享，易被情感打动 | 25% |
| 理性型 | 关注事实和数据，不易被营销影响 | 15% |

### 1.2 用户痛点

**核心问题：用户为什么讨厌广告？**

1. **内容无关**：广告与用户正在观看的内容毫无关系
   - 用户在看悬疑剧，突然插入食品广告
   - 用户对美妆不感兴趣，却被推送化妆品广告

2. **形式单一**：强制观看，无法选择
   - 无法跳过或快速获取关键信息
   - 广告内容冗长，核心信息被稀释

3. **需求不匹配**：广告推荐不精准
   - 平台只根据简单标签推送，未考虑用户真实需求
   - 同一广告对所有用户展示相同内容

4. **打断体验**：在关键时刻插入广告
   - 剧情高潮时被广告打断
   - 用户注意力被强制转移

**数据佐证：**
- 72%用户表示会跳过与自己无关的广告
- 65%用户认为现有广告"太长、太无聊"
- 仅15%用户能记住最近看过的广告内容

### 1.3 使用场景

**场景一：追剧用户**
- 小王，25岁，正在追一部悬疑剧
- 剧情紧张时，插入一段30秒的洗发水广告
- 传统体验：小王选择静音或切换标签页，完全忽略广告
- 精粹体验：小王点击"广告精粹"，AI识别他对"控油去屑"感兴趣，15秒内展示核心亮点，小王记住了产品卖点

**场景二：通勤用户**
- 小李，28岁，上班路上看短视频
- 每隔3分钟插入一段广告
- 传统体验：小李快速跳过，完全不看
- 精粹体验：AI识别小李是"碎片化观看"模式，自动提取最核心的1-2个亮点，5秒内完成展示

## 模块二：产品方案设计

### 2.1 产品概述

**一句话定义：**
广告精粹是一个AI驱动的广告体验优化系统，通过分析用户画像和广告内容，将传统广告转化为个性化的"亮点速览"，让用户在更短时间内获取更有价值的信息。

**核心价值：**
- 用户：从"被迫看广告"变为"主动获取信息"
- 平台：广告效果提升，用户留存率提高
- 广告商：精准触达目标用户，转化率提升

### 2.2 核心功能

**功能一：用户画像生成**
- 基于用户观看历史，自动提取13个维度的用户标签
- 包括：会员身份、内容偏好、观看行为、互动强度、当前情境等
- 每次随机生成用户画像，模拟不同用户群体

**功能二：AI亮点提取**
- 大模型分析广告内容，提取6-8个产品特性
- 根据用户画像，计算每个特性与用户的"共鸣系数"
- 从Top5特性中随机选择3个，增加多样性

**功能三：个性化文案生成**
- 大模型根据用户画像和产品特性，生成个性化文案
- 文案风格根据用户偏好自动调整（理性/感性/幽默等）
- 输出"用户友好特征名" + "推荐语"的组合

**功能四：拼贴式展示**
- 采用手账风格的便利贴布局
- 荧光笔高亮核心特征
- 2倍速播放广告，同步展示亮点

```
     ↓
展示拼贴式亮点（手账风格）
```

### 2.5 创新与差异化

**与现有方案对比：**

| 维度 | 传统广告 | 现有推荐 | 广告精粹 |
|------|---------|---------|---------|
| 用户体验 | 被动接受 | 简单标签匹配 | 深度心理分析 |
| 内容形式 | 完整视频 | 缩略图+文字 | 个性化亮点 |
| 精准度 | 低 | 中 | 高 |
| AI应用 | 无 | 基础推荐 | 深度理解+生成 |

**核心创新点：**

1. **7维度心理分析**：不只看用户"看了什么"，更分析用户"为什么看"
2. **共鸣系数算法**：量化用户与产品特性的匹配程度
3. **动态亮点选择**：Top5随机选3，增加多样性
4. **消费心理学文案**：根据用户心理特征生成文案，而非简单描述产品

## 模块三：AI原生能力说明

### 3.1 AI核心能力

**能力一：用户画像理解**
- 输入：用户观看历史、行为数据
- 输出：13维度用户标签
- AI作用：理解用户深层偏好，而非简单分类

**能力二：产品特性分析**
- 输入：产品文档、功能描述
- 输出：7维度心理得分（掌控感、效率、归属、感官、关爱、经济、自我）
- AI作用：从消费心理学角度理解产品价值

**能力三：个性化文案生成**
- 输入：用户画像、产品特性、共鸣系数
- 输出：个性化亮点文案
- AI作用：根据用户心理特征调整语言风格和表达方式

### 3.2 AI如何解决痛点

**痛点：广告内容无关**
- AI解决方案：分析用户心理偏好，提取与用户最相关的产品特性
- 效果：用户看到的不再是"随机广告"，而是"为自己定制的信息"

**痛点：形式单一**
- AI解决方案：将冗长广告压缩为3个个性化亮点
- 效果：从30秒被动观看 → 15秒主动获取

**痛点：需求不匹配**
- AI解决方案：7维度心理分析，精准匹配用户需求
- 效果：广告从"打扰"变为"服务"

**如果没有AI：**
- 无法深度理解用户心理偏好
- 无法生成个性化文案
- 无法实现精准匹配
- 产品退化为简单的标签推荐，失去核心价值

### 3.3 AI技术方案

**模型选择：** DeepSeek V4 Flash

**API调用流程：**
```
1. 构建Prompt
   - 用户画像（13维度）
   - 产品特性（Top3）
   - 权重分析（前3位主导维度）
   - 文案要求（字数、风格、禁忌）
   
2. 调用API
   - 端点：https://api.deepseek.com/chat/completions
   - 模型：deepseek-v4-flash
   - 温度：0.7
   - 最大token：1500
   
3. 解析返回
   - 提取results数组
   - 获取friendlyLabel和copy
   - 处理异常情况
```

**Prompt设计原则：**
- 消费心理学角度，转化而非描述
- 禁止直接提及用户画像标签
- 每条文案必须回答"这和我有什么关系"
- 根据用户特征调整语言风格

## 模块四：加分项

### 4.1 落地可行性

**技术实现路径：**
- 前端：Next.js + React + Tailwind CSS
- 后端：Next.js API Routes
- AI：DeepSeek API
- 视频：video.js播放器

**开发节奏：**
- 第一阶段（1周）：基础框架、用户画像系统
- 第二阶段（1周）：AI集成、亮点生成
- 第三阶段（1周）：UI优化、测试调优

**所需资源：**
- AI API调用费用：约$0.01/次
- 服务器：Vercel免费额度足够
- 视频资源：使用公开广告素材

### 4.2 商业化思考

**盈利模式：**
1. **SaaS服务**：向视频平台提供广告优化API
2. **效果分成**：按广告转化效果收取佣金
3. **数据服务**：提供用户画像分析报告

**目标市场：**
- 视频平台（腾讯、爱奇艺、优酷）
- 广告代理商
- 品牌方

**竞争优势：**
1. 深度心理分析，非简单标签匹配
2. 个性化文案生成，非模板化内容
3. 实时响应，用户体验友好
4. 三方共赢，可持续发展

**最大风险与挑战：**

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| AI生成内容质量不稳定 | 用户体验波动 | 增加人工审核、优化Prompt |
| 用户隐私担忧 | 合规风险 | 使用匿名化数据、明确告知用户 |
| 广告商接受度 | 商业化难度 | 先做效果验证，再推广 |
| 技术成本 | 盈利压力 | 优化API调用、使用更经济的模型 |
